# Решение проблемы с подключением к API серверу

## Проблема: "Network request failed" в Android эмуляторе

### Решение 1: Использовать 10.0.2.2 (для эмулятора)

1. Откройте `config/api.js`
2. Убедитесь, что для Android используется: `http://10.0.2.2:3001`
3. Перезапустите приложение

### Решение 2: Использовать реальный IP (для реального устройства или если 10.0.2.2 не работает)

1. Откройте `config/api.js`
2. Измените строку для Android на: `return 'http://192.168.0.20:3001';`
3. Убедитесь, что устройство/эмулятор и компьютер в одной Wi-Fi сети
4. Перезапустите приложение

### Решение 3: Проверить настройки сети эмулятора

Если используете Android Studio эмулятор:

1. Откройте настройки эмулятора
2. Settings → Network → Advanced
3. Убедитесь, что выбрана правильная сеть
4. Перезапустите эмулятор

### Решение 4: Использовать adb port forwarding

```bash
# Пробросить порт 3001 с хоста в эмулятор
adb reverse tcp:3001 tcp:3001

# Затем в config/api.js используйте:
return 'http://localhost:3001';
```

### Решение 5: Проверить файрвол

Убедитесь, что файрвол не блокирует порт 3001:

```bash
# Проверить, слушает ли сервер на всех интерфейсах
netstat -tuln | grep 3001
# Должно быть: 0.0.0.0:3001
```

### Текущая конфигурация

Сейчас в `config/api.js` используется `http://10.0.2.2:3001` для Android.

Если это не работает:
1. Попробуйте `http://192.168.0.20:3001`
2. Или используйте adb port forwarding с `http://localhost:3001`
